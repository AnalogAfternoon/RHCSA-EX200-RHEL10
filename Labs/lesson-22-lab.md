# Lesson 22 Lab — Troubleshooting RHEL

Complete the following tasks. Do not refer to your notes.

1. Display all units that have failed to start.

2. View all error messages from the current boot using the journal.

3. View kernel messages related to hardware and boot using `dmesg`.

4. Describe the step-by-step procedure to reset a lost root password using `rd.break`. (Write the steps — no command to run now.)

5. Check and repair a filesystem on `/dev/sdb1` (unmount it first if needed).

6. Use `sos report` to generate a system diagnostic report.

7. View the NetworkManager logs to troubleshoot a network issue.

8. Display the last 50 lines of `/var/log/messages` and filter for the word "error".
